# Отчет по заданию: UML-диаграммы для проекта Firamir

## 1. Предметная область

Разрабатываемый продукт `Firamir` — настольная информационная система для ЦППМСП «Здоровье», предназначенная для:

- авторизации администратора и разграничения доступа;
- ведения картотеки пациентов;
- регистрации приемов пациентов специалистами;
- просмотра журнала приемов;
- формирования аналитических и статистических отчетов в формате PDF.

Система работает с базой данных `SQL Server` (`testjob`) через `Entity Framework 6` (основная работа с данными) и частично через `ADO.NET` (`SqlConnection`) для генерации отчетов.

## 2. Подход к выполнению задания

Для моделирования использован `PlantUML` как текстовый инструмент UML. Подход включал три шага:

1. Выделение ключевых акторов, сущностей и бизнес-сценариев по существующему коду проекта (`MainWindow`, `MainInterface`, `Cart`, `NewCard`, `NewJoin`, `Journal`, `Entities`, `Reports`).
2. Построение набора диаграмм, покрывающих статическую и динамическую части системы.
3. Генерация программного кода на основе диаграммы классов (доменная модель C#).

## 3. Реализованные диаграммы

Созданы следующие файлы:

- `uml_use_case.puml` — диаграмма вариантов использования (Use Case);
- `uml_class.puml` — диаграмма классов (Class Diagram);
- `uml_sequence_login.puml` — диаграмма последовательности авторизации;
- `uml_state_auth.puml` — диаграмма состояний процесса входа;
- `uml_activity_visit.puml` — диаграмма деятельности сценария создания приема.

Диаграммы раскрывают:

- **статическую сущность**: структуры классов, связи и зависимости;
- **динамическую сущность**: последовательность сообщений, жизненный цикл авторизации и поток действий при создании приема.

## 4. Генерация программного кода

По диаграмме классов создан файл:

- `generated_domain_classes.cs`

Он содержит базовые классы предметной области:

- `User`, `Role`;
- `PatientCard`, `PatientVisit`.

Эти классы могут использоваться как стартовая доменная модель для дальнейшей реализации бизнес-логики и/или слоя доступа к данным.

## 5. Демонстрация процесса генерации (для презентации)

В презентации рекомендуется показать:

1. Исходный текст одной диаграммы `PlantUML` (например, `uml_class.puml`).
2. Результат визуализации диаграммы (PNG/SVG).
3. Переход от диаграммы классов к коду (`generated_domain_classes.cs`).
4. Краткий вывод: UML обеспечил формализацию требований и ускорил структурирование архитектуры решения.
