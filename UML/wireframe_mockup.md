# Прототип интерфейса (wireframe) для ВКР: Firamir

Ниже представлен прототип основных экранов системы в формате `Mermaid`.
Диаграмма показывает структуру экранов и пользовательские переходы.

```mermaid
flowchart TB
    %% --- Экран авторизации ---
    subgraph S1["Экран 1. Авторизация (MainWindow)"]
        A1["Логотип / Заголовок системы"]
        A2["Поле: Логин"]
        A3["Поле: Пароль"]
        A4["Кнопка: Войти"]
        A5["Сообщение об ошибке (при неверных данных)"]
        A2 --> A3 --> A4
        A4 -.ошибка.-> A5
    end

    %% --- Главное меню ---
    subgraph S2["Экран 2. Главное меню (MainInterface)"]
        M1["Кнопка: Картотека"]
        M2["Кнопка: Журнал приема"]
        M3["Кнопка: Новая карточка"]
        M4["Кнопка: Новый прием"]
        M5["Блок отчетов: выбор дат + кнопки PDF"]
    end

    %% --- Картотека ---
    subgraph S3["Экран 3. Картотека пациентов (Cart)"]
        C1["Поиск пациента"]
        C2["Таблица пациентов"]
        C3["Кнопка: Открыть карточку"]
        C4["Кнопка: Назад"]
        C1 --> C2 --> C3
    end

    %% --- Карточка пациента ---
    subgraph S4["Экран 4. Карточка пациента (NewCard/NewCard2)"]
        P1["Анкетные данные пациента"]
        P2["Справочники (выпадающие списки)"]
        P3["Кнопка: Сохранить"]
        P4["Кнопка: Назад"]
        P1 --> P2 --> P3
    end

    %% --- Новый прием ---
    subgraph S5["Экран 5. Новый прием (NewJoin)"]
        J1["Выбор пациента"]
        J2["Дата приема"]
        J3["Специалист / причина / заключение"]
        J4["Кнопка: Сохранить прием"]
        J5["Валидация обязательных полей"]
        J1 --> J2 --> J3 --> J4
        J4 --> J5
    end

    %% --- Журнал ---
    subgraph S6["Экран 6. Журнал приемов (Journal)"]
        G1["Фильтр по периоду"]
        G2["Таблица приемов"]
        G3["Кнопка: Печать / отчет"]
        G1 --> G2 --> G3
    end

    %% --- Переходы между экранами ---
    A4 -- "успешный вход" --> S2
    M1 --> S3
    M2 --> S6
    M3 --> S4
    M4 --> S5
    S3 -- "Открыть карточку" --> S4
    S5 -- "После сохранения" --> S2
    S4 -- "Назад" --> S2
    S6 -- "Назад" --> S2
```

## Комментарии к макету

- Переход из `Авторизации` в `Главное меню` выполняется только при корректных учетных данных.
- Экран `Главное меню` является центральным узлом навигации по системе.
- В `Картотеке` пользователь может найти пациента и открыть его карточку для редактирования.
- Экран `Новый прием` содержит обязательную проверку полей перед сохранением.
- Из `Журнала` доступны фильтрация данных и запуск печати/формирования отчета.

## Что показать в защите

- Логику переходов: `Вход -> Главное меню -> рабочие экраны`.
- Где выполняется ввод, где просмотр, где сохранение данных.
- Какие экраны связаны с отчетами и какие с первичным вводом данных.
